infer_cmap()#

Infers a suitable colormap from a wide candidate list based on a detailed analysis of the data’s statistical properties. Now, for each case, cycles through several candidate colormaps.

infer_norm()#

Infers the best normalization based on the statistical properties of the data.

TwoSlopeNorm: vcenter, vmin, vmax,

Useful when mapping data with an unequal rates of change around a conceptual center, e.g., data that range from -2 to 4, with 0 as the midpoint.

CenteredNorm: vcenter, halfrange, clip

Useful when mapping symmetrical data around a conceptual center e.g., data that range from -2 to 4, with 0 as the midpoint, and with equal rates of change around that midpoint.

LogNorm: vmin, vmax, clip

SymLogNorm: vmin, vmax, clip

setter linthresh Since the values close to zero tend toward infinity, there is a need to have a range around zero that is linear. The parameter linthresh allows the user to specify the size of this range (-linthresh, linthresh).

AsinhNorm: vmin, vmax, clip

setter linear_width The inverse hyperbolic sine scale is approximately linear near the origin, but becomes logarithmic for larger positive or negative values. Unlike the SymLogNorm, the transition between these linear and logarithmic regions is smooth, which may reduce the risk of visual artifacts.

PowerNorm: gamma, vmin, vmax, clip

Linearly map a given value to the 0-1 range and then apply a power-law normalization over that range.

BoundaryNorm: boundaries, ncolors, clip=False, *, extend=’neither

Unlike Normalize or LogNorm, BoundaryNorm maps values to integers instead of to the interval 0-1.
