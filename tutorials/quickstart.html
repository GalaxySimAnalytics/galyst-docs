Quick-start#

Note

Galyst is built on top of pynbody and tangos. For advanced usage and deeper understanding, please refer to their documentation.

Galyst is designed for the analysis and management of galaxy simulation data, providing a unified framework for multi-dimensional galaxy property calculations, model fitting, and data management.

  • Data Management (galyst.core):

    • At the simulation level, tangos is used to organize and access data from multiple simulations, as well as to store analysis results.

    • At the raw data level, pynbody enables reading simulation data files and performing detailed calculations and analyses.

Galyst provides essential tools for calculating galaxy properties across multiple dimensions:

  • Computation (galyst.calculators):

    • 0D: Global properties such as mass, size, and angular momentum.

    • 1D: Radial profiles, including density, age, and metallicity.

    • 2D: Images, such as surface brightness and velocity fields.

    Galyst also interfaces with external tools like agama and galpy for orbital analysis, and skirt for radiative transfer to generate synthetic observations.

Flexible multi-dimensional model fitting capabilities are available:

  • Model Fitting (galyst.fitting):

    • 1D: Fit theoretical profiles, probability density functions, radial density profiles, and star formation history templates.

    • 2D: Perform multi-component model fitting with imfit, ellipse fitting with photutils, and Fourier decomposition.

    • 3D: Fit three-dimensional models, such as ellipsoid fitting.

Additionally, Galyst offers a collection of pre-defined computed properties and methods in galyst.properties.

Setting up Paths#

Galyst uses tangos to organize and access simulation data. In tangos, the TANGOS_SIMULATION_FOLDER environment variable should point to your simulation data directory, and TANGOS_DB_CONNECTION should point to your tangos database.

If these two environment variables are not set, the default paths are:

TANGOS_SIMULATION_FOLDER = ~/
TANGOS_DB_CONNECTION = ~/tangos_data.db

Set the environment variables as follows:

export TANGOS_SIMULATION_FOLDER=~/Simulation/sims
export TANGOS_DB_CONNECTION=~/tangos_data.db

Alternatively, before running Python, you can set them within your script:

import os

os.environ['TANGOS_SIMULATION_FOLDER'] = os.path.expanduser('~/Simulation/sims')
os.environ['TANGOS_DB_CONNECTION'] = os.path.expanduser('~/tangos_data.db')

Connecting to the Database#

Galyst automatically connects to the database and creates a new one if needed, initializing the permissions management system.

In [1]: from galyst.core import Simulation

If you connect to a database that does not exist, the system will automatically create a new database and set the current user as the database author. After running :python:`from galyst.core import Simulation`, you will see output similar to the following, indicating that a new database has been created and the creator has been set as the author:

[2025-07-06 14:16:42.024] < galyst.core.permissions > | INFO | First user detected. Setting as Author.
[2025-07-06 14:16:42.176] < galyst.core.permissions > | INFO |   - User 'yxi@yxi' has been set as the Author.
[2025-07-06 14:16:42.337] < galyst.core.permissions > | INFO | Connection successful. User: 'yxi', Roles: ['author']

If the database already exists, you will see similar output indicating a successful connection:

[2025-07-06 14:34:47.458] < galyst.core.permissions > | INFO | Connection successful. User: 'yxi', Roles: ['author']

Adding Simulations#

Before analyzing your data, you need to add your simulation to Galyst. Initially, there are no simulations present.

In [2]: print(Simulation.all_simulations())
[]

Here, we add a high-resolution zoom-in simulation. When you use the Simulation class, it first checks the tangos_data.db database to see if the simulation already exists. If it does not, it parses the specified path, creates a new simulation instance, and adds it to the database.

In [3]: sim = Simulation("~/Simulation/sims/TNG50_l7_Halo274_l11")

In [4]: print(sim)
Simulation(name=TNG50_l7_Halo274_l11, path=/home/yxi/Simulation/sims/TNG50_l7_Halo274_l11)

In [5]: print(Simulation.all_simulations())
[<Simulation("TNG50_l7_Halo274_l11")>]

The folder structure is as follows:

TNG50_l7_Halo274_l11/
└── output/
    ├── groups_000/
    ├── ...
    ├── groups_099/
    │   ├── fof_subhalo_tab_099.0.hdf5
    │   ├── ...
    │   └── fof_subhalo_tab_099.10.hdf5
    ├── snapdir_000/
    ├── ...
    └── snapdir_099/
        ├── snapshot_099.0.hdf5
        ├── ...
        └── snapshot_099.10.hdf5

When adding a simulation, Galyst will automatically extract some basic properties and store them in the database. You can view these properties with the following code:

In [6]: print(sim.properties)
{'approx_resolution_kpc': 0.29524657089737566, 'approx_resolution_Msol': 532431.5564841477, 'boxsize': 51668.149907040715, 'h': 0.6774, 'handler': 'galyst.core.input_handler.ArepoHDFSubfindInputHandler', 'object_renumber': 1}

Adding TimeSteps#

You can access all timestep files with the following code:

In [7]: print('Number of timestep files:', len(sim.all_timestep_files))
Number of timestep files: 100

In [8]: print('Last timestep file:', sim.all_timestep_files[-1])
Last timestep file: /home/yxi/Simulation/sims/TNG50_l7_Halo274_l11/output/snapdir_099/snap_099

You can view the timesteps currently stored in the database. The following code displays the timesteps for this simulation that are not yet stored in the database, so it returns an empty list.

In [9]: print(sim.timesteps)
[]

However, you can still access a TimeStep that has not yet been stored in the database by using a numerical index:

In [10]: timestep = sim[99]

In [11]: print(timestep)
<TimeStep 'TNG50_l7_Halo274_l11/output/snapdir_099/snap_099' z=0.00 t=13.80 Gyr>

Alternatively, you can access a specific time step using its unique string identifier from the timestep files. In some cases, this method may be more precise.

In [12]: timestep = sim['snap_099']  # or '099'

In [13]: print(timestep)
<TimeStep 'TNG50_l7_Halo274_l11/output/snapdir_099/snap_099' z=0.00 t=13.80 Gyr>

After accessing a TimeStep in this way, it will automatically be added to the simulation in the database.

In [14]: print(sim.timesteps)
[<TimeStep 'TNG50_l7_Halo274_l11/output/snapdir_099/snap_099' z=0.00 t=13.80 Gyr>]

Note

Note that “halos” here typically refer to objects identified by the Subfind algorithm, while “groups” represent larger structures usually found by the FOF algorithm.

Adding Halos#

Within a TimeStep, you can inspect information about objects at that particular snapshot. For example, you can view the halos at this time step. Since no objects have been added yet, this will return an empty list.

In [15]: print(timestep.halos())
[]

Next, let’s add a halo to this time step.

In [16]: timestep.add_halos(max_object=1)

In [17]: print(timestep.halos())
[<Halo 'TNG50_l7_Halo274_l11/output/snapdir_099/snap_099/halo_1' | NDM=1838428 Nstar=750955 Ngas=613128>]

Now, timestep.halos() contains a halo object. You can access it by index:

In [18]: halo = timestep.halos()[0]

In [19]: print(halo)
<Halo 'TNG50_l7_Halo274_l11/output/snapdir_099/snap_099/halo_1' | NDM=1838428 Nstar=750955 Ngas=613128>

Alternatively, you can access a halo by specifying the finder_id:

In [20]: print(timestep.halos(finder_id=0))
[<Halo 'TNG50_l7_Halo274_l11/output/snapdir_099/snap_099/halo_1' | NDM=1838428 Nstar=750955 Ngas=613128>]

In [21]: halo = timestep.halos(finder_id=0)[0]

The halo object is a galyst.core.SimulationObject that contains all the properties of the halo at this time step. You can view all the properties stored in the database for this object. Since nothing has been stored yet, the list will be empty:

In [22]: print(halo.keys())
[]

Next, you need to load the particle data for this halo into memory to perform further calculations and analyses. The variable halo_particles is a SimSnap class from pynbody.

In [23]: halo_particles = halo.load()

In [24]: print(halo_particles)
<ArepoHDFSnap "/home/yxi/Simulation/sims/TNG50_l7_Halo274_l11/output/snapdir_099/snap_099" len=3202512>

Calculating Basic Properties#

A suite of tools for processing particle data is provided in galyst.calculators. Before performing analysis, it is recommended to orient halo_particles face-on.

In [25]: from galyst import calculators

In [26]: trans = calculators.transformator.faceon(halo_particles)

You can then use other tools in galyst.calculators for further calculations and analysis. For example, galyst.calculators.basic provides functions for computing fundamental galaxy properties. For example, to compute the half-mass radius:

In [27]: Re = calculators.basic.RadiusContain2D(frac=0.5)

In [28]: print(
   ....:     "Half-mass radius (total):", Re(halo_particles),
   ....:     "\nHalf-mass radius (DM):", Re(halo_particles.dm),
   ....:     "\nHalf-mass radius (stars):", Re(halo_particles.s),
   ....:     "\nHalf-mass radius (gas):", Re(halo_particles.g),
   ....: )
   ....: 
Half-mass radius (total): 62.9724720596586 
Half-mass radius (DM): 67.59941239234715 
Half-mass radius (stars): 5.850830467864327 
Half-mass radius (gas): 64.86895606235284

Here, the frac parameter specifies the mass fraction used to compute the radius. Setting frac=0.5 returns the radius containing half of the total mass (i.e., the half-mass radius).

To compute the rotational energy fraction:

In [29]: krot = calculators.basic.RotationEnergyFraction()

In [30]: print(
   ....:     "Rotation energy fraction (total):", krot(halo_particles),
   ....:     "\nRotation energy fraction (DM):", krot(halo_particles.dm),
   ....:     "\nRotation energy fraction (stars):", krot(halo_particles.s),
   ....:     "\nRotation energy fraction (gas):", krot(halo_particles.g),
   ....: )
   ....: 
Rotation energy fraction (total): 0.35872570255796943 
Rotation energy fraction (DM): 0.32376409351966806 
Rotation energy fraction (stars): 0.6905072443781521 
Rotation energy fraction (gas): 0.722732837582208

Calculating 1D Profiles#

The galyst.calculators.profile module provides tools for calculating 1D profiles. For example, you can use the basic galyst.calculators.profile.Profile class:

In [31]: pr = calculators.profile.Profile(halo_particles, ndim=2, rmax=30, qweights='mass')

The ndim parameter specifies the dimensionality; here, 2 indicates that the profile is computed in projection. rmax sets the maximum radius for the profile, and qweights determines the weighting used when calculating percentiles.

Let’s examine the density profiles for different components:

In [32]: import pylab

In [33]: pylab.plot(pr['rbins'], pr['density'], label='Total')
Out[33]: [<matplotlib.lines.Line2D at 0x7dca75570b90>]

In [34]: pylab.plot(pr['rbins'], pr['star-density'], label='Stars')
Out[34]: [<matplotlib.lines.Line2D at 0x7dca7841db50>]

In [35]: pylab.plot(pr['rbins'], pr['gas-density'], label='Gas')
Out[35]: [<matplotlib.lines.Line2D at 0x7dca7841fb10>]

In [36]: pylab.plot(pr['rbins'], pr['dm-density'], label='Dark Matter')
Out[36]: [<matplotlib.lines.Line2D at 0x7dca78424410>]

In [37]: pylab.yscale('log')

In [38]: pylab.xlabel('Radius ' + f"$[{pr['rbins'].units.latex()}]$")
Out[38]: Text(0.5, 0, 'Radius $[\\mathrm{kpc}]$')

In [39]: pylab.ylabel('Density ' + f"$[{pr['density'].units.latex()}]$")
Out[39]: Text(0, 0.5, 'Density $[M_{\\odot}\\,\\mathrm{kpc}^{-2}]$')

In [40]: pylab.xlim(0, 30)
Out[40]: (0.0, 30.0)

In [41]: pylab.legend()
Out[41]: <matplotlib.legend.Legend at 0x7dca7942a6d0>

Here, the prefixes star-, gas-, and dm- specify different matter components.

Let’s also look at the distribution of gas temperature:

Out[42]: [<matplotlib.lines.Line2D at 0x7dca7807ef50>]

Out[43]: <matplotlib.collections.FillBetweenPolyCollection at 0x7dca78540410>

In [44]: pylab.yscale('log')

In [45]: pylab.xlabel('Radius '+ f"$[{pr['rbins'].units.latex()}]$")
Out[45]: Text(0.5, 0, 'Radius $[\\mathrm{kpc}]$')

In [46]: pylab.ylabel('Temp '+ f"$[{pr['gas-temp_50'].units.latex()}]$")
Out[46]: Text(0, 0.5, 'Temp $[\\mathrm{K}]$')

In [47]: pylab.xlim(0,30)
Out[47]: (0.0, 30.0)

Plotting Halo Images#

The galyst.calculators.image module provides tools for generating images of halos.

For example, to plot the gas surface density distribution:

In [49]: from galyst.calculators import image as im

In [50]: im_g = im.sph_projection(halo_particles.g, width='60 kpc', units='Msol kpc^-2')

In [51]: im_g.show()
Out[51]: <matplotlib.image.AxesImage at 0x7dca77d44910>

Gas Metallicity

In [52]: im_g_metals = im.sph_projection(halo_particles.g, quantity='metals', width='60 kpc')

In [53]: im_g_metals.show()
Out[53]: <matplotlib.image.AxesImage at 0x7dca77d09a50>

To plot the stellar luminosity image:

In [54]: im_s = im.star_luminosity_image(halo_particles, width='60 kpc')

In [55]: im_s.show()
Out[55]: <matplotlib.image.AxesImage at 0x7dca77b06750>

You can see a short bar in the center. Let’s examine its kinematic features:

In [56]: im_s_m = im.sph_projection(halo_particles.s, smooth_floor=0.1, units='Msol kpc^-2', width='5 kpc')

In [57]: im_s_v = im.sph_projection(halo_particles.s, smooth_floor=0.1, quantity='vrxy', width='5 kpc')

In [58]: image = im_s_v.show()

In [59]: im_s_m.show_contour(image)
Out[59]: <matplotlib.contour.QuadContourSet at 0x7dca77f4a550>
